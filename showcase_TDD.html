Test Driven Development with Pytest

I wanted to create a tool that simplifies my weekly meal planning by combining all the separate ingredients I need into one comprehensive list. For this project, I decided to use Pytest and follow the principles of Test-Driven Development (TDD), that is to say:
  1. Write a test
  2. Write the minimum amount of code required to pass that test
  3. Refactor that code to improve itâ€™s design and maintainability
  4. Repeat this cycle with a new test (until the application is complete)
At first I felt like the TDD approach was significantly slowing me down, but as I progressed through the project, I was pleased at how smoothly all the different components of the application came together. Additionally, I feel confident that if I wanted to expand or modify the functionality, it could be done quickly and simply, thanks to the modular design.
